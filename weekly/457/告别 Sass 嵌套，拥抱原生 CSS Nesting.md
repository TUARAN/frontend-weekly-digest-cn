# 告别 Sass 嵌套，拥抱原生 CSS Nesting

很多前端开发者第一次学会 Sass 嵌套时，都会有一种“终于不用重复写选择器了”的解脱感。它曾经很长一段时间都是样式工程化的重要工具，甚至影响了一代人组织 CSS 的方式。但现在原生 CSS Nesting 越来越成熟，问题就不再是“要不要嵌套”，而是“我们是否该重新思考过去那套嵌套习惯”。

这篇文章最有价值的地方，不只是介绍语法，而是它代表了一种迁移心态：当平台原生开始接住曾经依赖预处理器的能力，我们不能只做机械替换，更应该回头看一眼，哪些写法其实是历史阶段产物，哪些又仍然值得保留。

原生 Nesting 的出现，未必会让 Sass 立刻失去价值，但它确实在提醒我们，CSS 正在重新变成一门更完整、更有自信的语言。很多过去“必须借助工具”的理由，正在慢慢缩小。

所以这篇文章值得读，不只是为了写更现代的 CSS，也为了重新理解：当平台成长后，我们该如何把历史工具链的习惯一点点重新校准。
