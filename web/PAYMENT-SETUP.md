# 支付接入说明

本站采用「静态导出 + 手动对账」的轻量支付流程，避免维护商户后端。用户在 `/order?plan=xxx` 页面扫码付款，微信联系 `atar24` 开通对应权益。

## 一、替换收款码

把你的真实收款码覆盖到以下路径（保留同名，推荐 PNG，透明背景优先）：

- `public/wechat-pay-qr.png` — 微信支付收款码
- `public/alipay-qr.png` — 支付宝收款码

当前仓库里放的是占位 SVG（`wechat-pay-qr.svg` / `alipay-qr.svg`），`/order` 页面会优先加载 PNG，PNG 不存在时回落到 SVG。

### 获取商户码

- 微信：微信 → 我 → 服务 → 收付款 → 二维码收款 → 保存收款码
- 支付宝：支付宝 → 收钱 → 保存收款码

> 建议开启「到账语音播报」和「收款备注」，方便核对订单。

## 二、订单号规则

`/order` 页面提交后会生成订单号，格式：

```
FN + YYYYMMDD + HHMM + 4位随机
例：FN202604221932-5821
```

用户付款时被要求把订单号填入付款备注。你在微信/支付宝看到到账消息时，凭订单号找到对应用户。

## 三、订单收集

由于当前是静态导出部署（`output: "export"`），`/api/order` 路由不会真正落库。订单在两个地方保留：

1. **用户本地浏览器 localStorage**：key 为 `fn_last_order`，包含姓名、微信、邮箱、套餐。
2. **用户主动联系**：done 页面引导用户加微信 `atar24`，把订单号发给你。

如果需要服务端收集，请把站点迁移到 Vercel / Node server，去掉 `output: "export"`，`/api/order/route.ts` 就会生效，可以接 Notion / Airtable / 自建数据库。

## 四、开通权益的人工流程（建议）

1. 在微信收到到账 + 用户发来订单号。
2. 记录到你的 Notion/表格（字段：订单号 / 姓名 / 微信 / 邮箱 / 套餐 / 付款时间 / 金额）。
3. 年费会员：拉进会员微信群、发送会员站点链接。
4. 1v1：发问卷链接、约 3 次连麦时间。

## 五、升级到自动发货（可选）

当每月订单 > 30 单时再考虑：

- 切换为带 Node server 的部署（Vercel / 自建）。
- 用微信 Native 扫码支付或支付宝 Precreate API（都需要营业执照开通商户号）。
- 或接入 Stripe + 支持微信/支付宝（海外商户更容易）。

现阶段不建议提前优化，手工对账反而让你离用户更近。
