# 原生 JSON Modules 终于成真

JSON 模块这件事，前端其实等了很多年。过去我们当然一直都能“导入 JSON”，但往往靠的是打包器或运行时特性兜底，而不是平台层真的把它作为一等模块类型来处理。原生 JSON Modules 的真正意义，不在于以后少写几行配置，而在于模块系统终于在一个很常见的资源类型上补齐了标准化路径。

这类能力看起来不够戏剧化，因为开发者早就习惯“反正能用”。可一旦你把视角放到平台演进，会发现这种“终于变得理所当然”的过程恰恰最重要。因为现代前端真正难受的，并不是缺某个新 API，而是同一件基础事情在浏览器、构建器、运行时之间总要靠不同魔法打补丁。

原生 JSON Modules 的落地，意味着前端模块生态又少了一块长期依赖工具默契的地带。平台能力一旦变得更统一，工程复杂度才会真的开始下降。很多时候，最值得高兴的更新不是给你更多超能力，而是终于把老问题做成了默认正确。
