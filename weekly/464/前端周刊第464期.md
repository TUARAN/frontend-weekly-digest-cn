💬 **推荐语**

这一期的主题，如果用一个词概括，就是「**回归平台**」——不是倒退，而是前端在经历框架狂热之后，重新开始认真审视浏览器和 CSS 本身的能力边界。

CSS 栏目是这期最密集的板块。从 Josh Comeau 系统讲解 scroll-driven animations，到 CSS 原生随机性和 `sibling-index()` 实现错位动画，再到 `hypot()` 这样的新函数——这些不再只是「CSS 小技巧」，而是平台层在持续把原本必须依赖 JavaScript 的能力收归到声明式语言里。当你意识到不用一行 JS 就能实现滚动驱动的复杂动效时，很多项目的架构选择会变得完全不同。与此同时，Tailwind 的争议也没有停——那篇「I Don't Like Tailwind」不是情绪发泄，而是提醒我们：**工具选择从来不是纯技术问题，团队协作成本、可维护性和代码可读性同样重要。**

JavaScript 和 Web Dev 栏目则在另一个维度呼应了这个主题。「Rendering Is a Browser Decision, Not a JavaScript One」一文直接把渲染的主动权交还给浏览器，这对习惯了「一切都在 JS 里解决」的开发者来说是必要的认知校准。React Compiler 十八个月回顾和 React 乱序 Streaming 的深度解析，则从框架层面告诉我们：**真正好的抽象，不是让你忘记平台，而是让你更高效地驾驭平台。**

## 🗂 本期精选目录

### 🧭 Web Dev

* [SVG from Scratch](https://www.carmenansio.com/articles/svg-from-scratch)：从零开始理解 SVG，不是罗列属性，而是帮你建立 SVG 的思维模型。
* [More Than a Portfolio: Building a Scroll-Driven 3D World with Something to Say](https://tympanus.net/codrops/2026/04/28/more-than-a-portfolio-building-a-scroll-driven-3d-world-with-something-to-say/)：用滚动驱动的 3D 场景来讲述故事，技术和表达融合得非常漂亮。
* [Automate the browser from the terminal with Chrome DevTools MCP CLI](https://joanleon.dev/en/chrome-devtools-mcp-cli/)：把 Chrome DevTools 的能力搬到终端，用 MCP 协议实现浏览器自动化，对工具链敏感的人值得关注。
* [Three stoic principles for better web accessibility](https://piccalil.li/blog/three-stoic-principles-for-better-web-accessibility/)：用斯多葛哲学的三条原则来审视无障碍——视角独特，但结论很务实。
* [Designing Stable Interfaces For Streaming Content](https://www.smashingmagazine.com/2026/05/designing-stable-interfaces-streaming-content/)：流式输出正在成为 UI 常态，这篇文章讲的是如何让界面在这种不确定性中保持稳定。
* [Preserving DOM Changes Across Live Reloads](https://kittygiraudel.com/2026/05/01/preserving-dom-changes-across-live-reloads/)：热重载时保留 DOM 状态，解决的是一个日常但一直被忽视的开发体验问题。
* [Canvas-ing the Web](https://meyerweb.com/eric/thoughts/2026/04/27/canvas-ing-the-web/)：Eric Meyer 的思考——当越来越多的网站选择用 Canvas 渲染整个页面，Web 的本质在发生什么变化？
* [Server-Driven UI in 22 lines of TypeScript](https://neciudan.dev/implementing-server-driven-ui)：用极简代码演示 Server-Driven UI 的核心逻辑，适合想快速理解这个概念的人。
* [pnpm 11.0](https://pnpm.io/blog/releases/11.0)：pnpm 11.0 发布，性能和包管理策略上都有值得关注的改进。

### 🎨 CSS

* [Scroll-Driven Animations: Exploring the majestic new animation-timeline API](https://www.joshwcomeau.com/animation/scroll-driven-animations/)：Josh Comeau 的深度讲解，把 scroll-driven animations 的 API 设计、使用场景和最佳实践讲得清清楚楚。如果你想入门这个方向，这是目前最好的起点。
* [Warning: containment breach in cascade layer!](https://dbushell.com/2026/04/15/containment-breach-in-cascade-layer/)：`@layer` 的 containment 不是绝对的，这篇文章揭示了级联层在实际使用中可能的「泄漏」场景。
* [hypot() in CSS](https://css-tricks.com/almanac/functions/h/hypot/)：CSS 新增的 `hypot()` 函数，虽然不是高频使用的功能，但它是 CSS 数学能力持续扩展的信号。
* [The Importance of Native Randomness in CSS](https://css-tricks.com/the-importance-of-native-randomness-in-css/)：CSS 原生随机数——这可能是今年 CSS 最被低估的新能力，一旦可用，大量的「随机视觉效果」不再需要 JS 参与。
* [Let's Use the Nonexistent ::nth-letter Selector Now](https://css-tricks.com/using-nonexistent-nth-letter-selector-now/)：一个「还不存在」但已经有了 polyfill 思路的选择器，展示了 CSS 社区的创造力和对未来的前瞻。
* [Native CSS stagger animations with sibling-index()](https://blog.logrocket.com/native-css-stagger-sibling-index/)：用 `sibling-index()` 实现原生的 CSS 错位动画，告别 JS 驱动的 stagger 效果。
* [I Don't Like Tailwind. Sorry Not Sorry](https://dev.to/freshcaffeine/i-dont-like-tailwind-sorry-not-sorry-50b5)：坦诚的 Tailwind 批评，不回避争议，讨论了实用优先的 CSS 方案在可维护性和代码可读性上的代价。

### 💡 JavaScripts

* [The React Compiler at Eighteen Months: The Arc, the Debates, and What's Next](https://saschb2b.com/blog/react-compiler-year-in-review)：React Compiler 发布一年半的真实回顾——它改变了什么、争议在哪里、接下来会走向何方。
* [Accessibility in React: Common Mistakes and How to Fix Them](https://certificates.dev/blog/accessibility-in-react-common-mistakes-and-how-to-fix-them)：React 中无障碍实现的常见错误和修复方案，每个 React 开发者都应该过一遍这份清单。
* [Next.js Link as a Button](https://kittygiraudel.com/2026/05/02/nextjs-link-as-a-button/)：一个看似简单但很容易踩坑的问题——把 Next.js 的 `Link` 当成按钮用，可访问性和语义化的细节不容忽视。
* [Rendering Is a Browser Decision, Not a JavaScript One](https://www.marshateo.com/writing/rendering)：核心观点直击要害——渲染最终是由浏览器决定的，JavaScript 框架只是请求者。理解这一点，很多性能优化和 SSR 策略的选择会清晰很多。
* [How React streams UI out of order and still manages to keep order](https://inside-react.vercel.app/blog/how-react-streams-ui-out-of-order)：深入 React Streaming SSR 的乱序传输机制，讲清楚了 React 如何在乱序输出中保持最终 UI 的正确性。

## 结语

这一期最值得被记住的，不是某个具体的 API 或工具更新，而是一个逐渐清晰的趋势：**CSS 正在以前所未有的速度吸收原本属于 JavaScript 的能力，而 React 生态则在反过来更深入地理解浏览器本身。**

无论是 scroll-driven animations、`sibling-index()` 还是 CSS 原生随机数，这些能力都在回答同一个问题——「能不能不写 JS 就做到？」。而对前端工程师来说，知道什么时候该用平台能力、什么时候该用框架能力，正是从「会用工具」走向「会做判断」的关键一步。
