# 别把单词拆成单个字母：可访问性警告

原文：https://adrianroselli.com/2026/02/you-know-what-just-dont-split-words-into-letters.html

作者：Adrian Roselli

发布时间：2026-02-05

这篇文章是作者对“把一个单词拆成一个个字母节点来做动画/排版”的再次提醒：这类做法会在多种读屏器 + 浏览器组合下严重破坏可访问性（把词读成一串字母，甚至干脆读不到）。

作者强调：这不是新问题——2012 年就有人用类似做法逐字母调字距（kerning），后来 AWWWards、BeeLine Reader 等也出现过类似实现。2026 年他再次写这篇，是因为 GSAP 的 SplitText 插件宣称“支持屏幕阅读器”，但实测并不成立。

## 问题来自哪里：逐字母 DOM 与读屏

GSAP 的 Demo 在“Characters”模式下，会把一句话拆成大量子元素，每个字母一个节点（示意）：

```html
<div aria-hidden="true" style="display: block;">
  <div aria-hidden="true" style="display: inline-block;">B</div>
  <div aria-hidden="true" style="display: inline-block;">r</div>
  <div aria-hidden="true" style="display: inline-block;">e</div>
  <!-- ...大量字母节点... -->
</div>
```

从视觉上看，这很适合做逐字动画；但对读屏器来说，经常意味着：

- 读成 “B r e a k …” 这种逐字母拼读
- 或者在某些组合下读不到原本的完整文本

## “用 ARIA 兜底”并不可靠

GSAP 的文档示例会建议用 `aria-label` 或类似方式给整段文字提供替代文本。

作者指出一个常被忽略的细节：

- `<div>` 在 HTML AAM 中映射到 ARIA 的 `generic` role
- `generic` role **不允许作者提供可访问名称**（name-from prohibited）
- 因此在很多情况下，对 `<div>` 使用 `aria-label` 是被禁止/无效的

参考：

- HTML AAM 中 `<div>` 对应 `generic`：https://www.w3.org/TR/html-aam-1.0/#el-div
- `generic` 的命名限制（namefromprohibited）：https://w3c.github.io/aria/#namefromprohibited
- `aria-label` 规则：https://w3c.github.io/aria/#aria-label

即便你选了允许 `aria-label` 的元素（如 heading），也仍有风险：

- `aria-label` 不一定会随页面语言自动翻译
- 可能触发 WCAG 2.5.3（Label in Name）等一致性问题

相关阅读：

- `aria-label` 不会自动翻译：https://adrianroselli.com/2019/11/aria-label-does-not-translate.html
- WCAG 2.5.3：https://www.w3.org/WAI/WCAG22/Understanding/label-in-name.html

## 实测结果（是/否）

作者把结果简化成二元判断（能否正确暴露/朗读文本）。

| 组合 | 是否可用 |
| --- | --- |
| NVDA / Firefox | ✅ 是 |
| JAWS / Chrome | ❌ 否 |
| Narrator / Edge | ❌ 否 |
| VoiceOver macOS / Safari | ❌ 否 |
| Orca / Firefox | ❌ 否 |
| TalkBack / Chrome | ✅ 是 |
| TalkBack / Firefox | ❌ 否 |
| VoiceOver iPadOS / Safari | ❌ 否 |

（原文包含多段录屏视频与字幕文件链接，用于佐证不同平台读屏行为；这里省略视频段落，只保留结论表。）

## 给作者的建议（以及 Bug 报告）

这篇文章首先是对“实现方/使用方”的警告：在读屏可访问性上，**不要依赖**“逐字母拆分 + ARIA 兜底”这条路。

作者也向 GSAP 提交了 issue，要求在 SplitText 页面明确风险与限制：

- https://github.com/greensock/GSAP/issues/642

在问题解决前，他强烈建议避免使用 SplitText（或任何同类逐字母拆分方案）。

## 结语

如果你为了字距、渐变、动画等效果而“必须”把单词拆成字母——作者的答案是：你并不需要。换一种实现方式。

一个更现实的判断是：就连宣称“读屏支持”的团队都可能做错；如果你没有覆盖所有读屏器/浏览器/平台/语音引擎组合并持续回归测试的能力，那么你大概率也会做错。

## 当天更新：另一个常见坑

作者还补充提到另一个案例：有的页面会把文本放在一个 `<span>` 里，但用 `aria-label` 替换可访问名称——同样可能带来语言/一致性/规则适配问题。

这也从侧面说明：可访问性需要专业与持续测试，而不是“加个 ARIA 属性就完事”。
