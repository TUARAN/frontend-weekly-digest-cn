💬 **推荐语**

**React 生态的"反思潮"来了。** 本期有一篇非常有代表性的文章——一位开发者详细记录了为何从 Next.js + RSC 架构退回到普通 SPA + 独立后端的决策过程。这不是一篇"反 Next.js"的情绪发泄，而是有具体踩坑细节的架构复盘：**Server Components 的数据绑定耦合、RSC 对构建链路的限制、以及 ISP 边界模糊带来的维护成本**。值得每一个在"全栈框架"路线上投入了真金白银的团队认真读一读。

**CSS 正在吃掉动画库的饭碗，这次是真的。** Josh Comeau 本期出手，深度对比了 CSS 关键帧、JS requestAnimationFrame、以及 Motion/GSAP 等动画库的性能差异。结论并不简单——**CSS 动画跑在独立线程，JS 动画跑在主线程**，这道理大家都懂，但文章指出了一个更微妙的点：Motion 之所以能"逃课"，是因为它底层用了 Web Animations API，也跑在独立线程。而未来 CSS 的 `linear()` 函数、Scroll-driven Animations 等特性，正在让"不需要 JS 的动画"从口号变成现实。**如果你还在为"该不该上动画库"纠结，这篇文章能帮你省掉至少一次技术评审。**

**跨文档视图过渡（Cross-Document View Transitions）进入"实战深水区"。** CSS-Tricks 本期的第二部分，专门解决"教程里只有单个 hero 图片过渡，真实项目里却有 200 张卡片"的扩展性问题。核心亮点是 `view-transition-class` 这个被低估的属性——它让你用一条 CSS 规则控制数百个元素的过渡样式，而不需要为每个元素单独写选择器。**如果你已经在用 View Transitions API，或者准备用，这篇文章能帮你绕过"选择器墙"这个必然踩到的坑。**

## 🗂 本期精选目录

### 🧭 Web Dev

* [Declarative partial updates](https://developer.chrome.com/blog/declarative-partial-updates)：Chrome 新提案，允许开发者声明式地控制页面局部更新，减少全量重新渲染的性能浪费，值得关注 Baseline 进展
* [CSS vs. JavaScript: Exploring the performance implications of different animation strategies](https://www.joshwcomeau.com/animation/css-vs-javascript/)：Josh Comeau 深度测评 CSS/JS 动画性能差异，结论是 CSS 跑独立线程、JS 跑主线程，但 Motion 靠 WAAPI "逃课"——**动画库选型必读**
* [Maybe Don't Rely on Google's "Modern Web Guidance"](https://adrianroselli.com/2026/05/maybe-dont-rely-on-googles-modern-web-guidance.html)：Adrian Roselli 犀利点评 Google 的"现代 Web 开发指南"，指出其中可访问性和语义化方面的误导性建议，**不是所有来自大厂的最佳实践都值得照搬**
* [TypeScript Is Not a Security Boundary](https://jsdev.space/typescript-security-guide/)：一文厘清 TypeScript 类型系统与运行时安全的关系——**类型注解不是输入校验，生成后的 JS 里类型信息全没了**，安全边界必须在运行时建立
* [Why I Walked Back from Next.js and RSC to a Plain SPA and a Separate Backend](https://dev.to/devrayat000/why-i-walked-back-from-nextjs-and-rsc-to-a-plain-spa-and-a-separate-backend-3ibo)：一位开发者的架构复盘——从 Next.js + RSC 退回普通 SPA + 独立后端，**RSC 的数据耦合、构建限制和 ISP 边界模糊是三大痛点**，每一个都是真金白银换来的教训
* [What's new in Astro — May 2026](https://astro.build/blog/whats-new-may-2026/)：Astro 5.7 发布，增量内容更新（ICU）让大型多语言站点构建速度大幅提升，Actions 正式稳定——**静态站点生成器的 DX 军备竞赛还在继续**

### 🎨 CSS

* [Algorithmic Theming Engines: Building Self-Correcting Color Systems With contrast-color()](https://www.smashingmagazine.com/2026/05/building-self-correcting-color-systems-contrast-color/)：Smashing Magazine 深度文章，介绍 CSS `contrast-color()` 函数如何自动计算可读性前景色，**配合 OKLCH 色彩空间，可以构建"不会翻车的"动态主题系统**，设计 token 工作流的重大升级
* [Bending a Straight Line using Modern CSS](https://css-tip.com/bending-line/)：用纯 CSS 实现线条弯曲效果，无需 SVG path，**`border-radius` 配合 `transform: rotate()` 的巧妙组合**，小技巧解决大问题
* [The Fundamentals and Dev Experience of CSS @function](https://frontendmasters.com/blog/the-fundamentals-and-dev-experience-of-css-function/)：CSS `@function` 规则的最新进展，允许在样式表中定义可复用函数，**离"CSS 变成一门编程语言"又近了一步**，但 DX 设计值得仔细审视
* [Managing Anchor Associations With Data Attributes and Advanced attr()](https://frontendmasters.com/blog/managing-anchor-associations-with-data-attributes-and-advanced-attr/)：CSS Anchor Positioning 进阶——用 `attr()` 函数动态管理锚点关联，**弹出层定位的终极方案正在成型**，Firefox 支持落地后可直接上生产
* [Revealing Text With CSS letter-spacing](https://css-tricks.com/revealing-text-with-css-letter-spacing/)：用 `letter-spacing` 动画实现文字逐字揭示效果，**GPU 友好的纯 CSS 动画方案**，比 JS 打字机效果性能更好
* [Cross-Document View Transitions: Scaling Across Hundreds of Elements](https://css-tricks.com/cross-document-view-transitions-part-2/)：CSS-Tricks 深度实战——跨文档视图过渡的扩展性问题。**用 `view-transition-class` 一条规则替代数百个选择器**，是近期 CSS 领域最实用的架构级技巧之一，配合 `pageswap`/`pagereveal` 事件实现完美的 SPA 级页面切换体验
* [What's missing in CSS layout](https://patrickbrosset.com/articles/2026-05-20-whats-missing-in-css-layout/)：Patrick Brosset（Firefox 前 DevTools 工程师）盘点 CSS 布局还缺少什么——**`justify-self`、`align-self` 在 Grid 以外的缺失、`gap` 在旧布局模式中的支持不完整**，以及真正的"对齐"原语的缺席，是一份来自引擎内部的布局需求清单
* [State of CSS 2026](https://survey.devographics.com/en-US/survey/state-of-css/2026)：年度 CSS 现状调查开启，**2026 年的数据将反映 CSS Nesting、`:has()`、Anchor Positioning 等特性真实采纳率**，填一填，看看自己和社区平均水平的差距

## 结语

**2026 年的前端周刊，信号越来越清晰了。** 一边是框架层的"反思"——Next.js RSC 的复杂性代价正在被更多团队公开讨论，SPA + BFF 的经典分层架构并没有过时，只是在新语境下被重新评估。另一边是浏览器原生的"进攻"——View Transitions API、`contrast-color()`、`@function`、Scroll-driven Animations……CSS 正在系统性地收回被 JS 库占据的领土。**这个趋势不是"JS 要完了"，而是"该用 CSS 的地方就用 CSS，该用 JS 的地方更好地用 JS"——边界正在重新划定，而能同时在两边游刃有余的开发者，价值会持续放大。**

**工具链层面，Rspack 的生产验证案例开始密集出现。** Yelp 的 6 分钟→70 秒是一个很强的信号，说明 Rust 编写的构建工具在工程大型项目上的优势已经从 benchmark 走向了真实业务。如果你所在的团队还在忍受 3 分钟以上的 dev server 启动时间，这篇文章是一个很好的立项依据。**但更重要的是：构建工具迁移的真正成本不在配置，而在生态系统的边缘**——loader 兼容性、HMR 行为差异、以及 CI 缓存策略的重新设计，Yelp 的文章在这些方面的诚实记录比性能数字更值钱。
