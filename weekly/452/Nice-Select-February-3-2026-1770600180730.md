          Core Architecture
          [#](https://nerdy.dev/nice-select#core-architecture)
        
       
This component builds on the new `appearance: base-select` foundation, combining native accessibility with sweet visual control. Progressive enhancement ensures the custom experience only loads on capable devices. 

  The examples in this post just look like regular select elements in Firefox and Safari. Firefox is [working on it though!](https://bugzilla.mozilla.org/show_bug.cgi?id=1944403)

**[`appearance: base-select`](https://developer.mozilla.org/docs/Web/CSS/appearance#base-select)**

unlocks [full customization](https://developer.chrome.com/blog/a-customizable-select) of native `<select>` elements while retaining browser controlled accessibility and keyboard navigation
**Conditional enhancement**

Only enables custom styling for devices with hover and fine pointer ([@media (hover)](https://developer.mozilla.org/docs/Web/CSS/@media/hover) and [(pointer: fine)](https://developer.mozilla.org/docs/Web/CSS/@media/pointer))
**JavaScript + CSS hybrid**

JS calculates an anchor offset for alignment while CSS handles all visual presentation and animation

        
          Alignment & Positioning
          [#](https://nerdy.dev/nice-select#alignment-&-positioning)
        
       
Anchor positioning plus some special JS sauce to write an inline CSS variable unlocks the picker to position near the selected option (obviously this can't work at the viewport edges). This creates a pleasant morphing effect that feels connected. 

Maybe I should have used [AIM](https://nerdy.dev/anchor-interpolated-morphing)?

                                                                                    

[CSS Anchor Positioning](https://developer.mozilla.org/docs/Web/CSS/CSS_anchor_positioning)

Picker uses [`anchor(start)`](https://developer.mozilla.org/docs/Web/CSS/Reference/Values/anchor) to position relative to the trigger button                                                                      
**Selected option alignment**

When dropdown opens, the currently selected option aligns vertically with the button text via `--_select-anchor-offset` [custom property](https://developer.mozilla.org/docs/Web/CSS/--*)  
**Syncs**
`transform-origin`

With `50% var(--_select-anchor-offset)` ensures scale animations line up better with the selected option's position                       
**Anchor fallbacks with**
[`position-try`](https://developer.mozilla.org/docs/Web/CSS/position-try-fallbacks)

`flip-block, flip-inline` automatically repositions the anchor position if the picker would overflow the viewport

        
          Animation & Transitions
          [#](https://nerdy.dev/nice-select#animation-&-transitions)
        
       
Scale effects with spring physics to scroll-driven option reveals, every interaction respects user motion preferences while providing rich feedback.

[`@starting-style`](https://developer.mozilla.org/docs/Web/CSS/Reference/At-rules/@starting-style)

Defines entry animation initial state for the popover                               
**Spring easing**

Uses [Open Props](https://open-props.style/#easing) `--ease-spring-3` for playful, physics-based motion on scale and rotation
**Discrete property transitions**

[display](https://developer.mozilla.org/docs/Web/CSS/display) and [overlay](https://developer.mozilla.org/docs/Web/CSS/overlay) use [`allow-discrete`](https://developer.mozilla.org/docs/Web/CSS/transition-behavior) for proper `::picker()` popover animation support
**Chevron rotation**

[`::picker-icon`](https://developer.mozilla.org/docs/Web/CSS/::picker-icon) rotates 180¬∞ on open with spring easing
**Scroll-driven option fade-in**

Options animate in from the bottom using [`animation-timeline: view()`](https://developer.mozilla.org/docs/Web/CSS/animation-timeline)
**Respects reduced motion**

Animations and transitions wrapped in [`@media (prefers-reduced-motion: no-preference)`](https://developer.mozilla.org/docs/Web/CSS/@media/prefers-reduced-motion)
**Button press feedback**

[Active state](https://developer.mozilla.org/docs/Web/CSS/:active) [scales](https://developer.mozilla.org/docs/Web/CSS/scale) down to `0.98` for tactile feedback
**Open state button feedback**

Select scales up to `1.04` when [`:open`](https://developer.mozilla.org/docs/Web/CSS/:open), scaling back down on close with a soft bounce

        
          Theming & Color Handling
          [#](https://nerdy.dev/nice-select#theming-&-color-handling)
        
       
Automatic light/dark adaptation leverages system colors and modern color functions for seamless theme switching. Definitely check the support for light/dark forced colors too, a really great way to debug spacing and skeletal aspects of the UI.

[`color-scheme: light dark`](https://developer.mozilla.org/docs/Web/CSS/color-scheme)

Enables automatic system color adaptation              
[`light-dark()` function](https://developer.mozilla.org/docs/Web/CSS/color_value/light-dark)

Used for backgrounds that need different light/dark values
[System colors](https://developer.mozilla.org/docs/Web/CSS/system-color)

`Canvas`, `CanvasText`, `Highlight`, `HighlightText` for semantic, colors
[`color-mix()`](https://developer.mozilla.org/docs/Web/CSS/color_value/color-mix)

Creates semi-transparent overlays and borders
**Data URI icons**

Chevron and checkmark icons embedded as SVG [data URIs](https://developer.mozilla.org/docs/Web/URI/Schemes/data) with variants for each color scheme
**Forced colors support**

[`@media (forced-colors: active)`](https://developer.mozilla.org/docs/Web/CSS/@media/forced-colors) provides high-contrast mode adaptations (even the sticky scroll state respects this!)

        
          Layout & Spacing
          [#](https://nerdy.dev/nice-select#layout-&-spacing)
        
       
Logical properties enable true internationalization with automatic RTL support, while text-box trim delivers pixel-perfect vertical alignment. Custom properties create a flexible design token system.

[CSS Custom Properties](https://developer.mozilla.org/docs/Web/CSS/--*) **design tokens**

All spacing, sizes, and timing values defined as `--_select-*` variables
[Logical properties](https://developer.mozilla.org/docs/Web/CSS/CSS_logical_properties_and_values) **throughout**

[`inline-size`](https://developer.mozilla.org/docs/Web/CSS/inline-size), [`block-size`](https://developer.mozilla.org/docs/Web/CSS/block-size), [`inset-block-start`](https://developer.mozilla.org/docs/Web/CSS/inset-block-start), [`margin-inline`](https://developer.mozilla.org/docs/Web/CSS/margin-inline), [`padding-block`](https://developer.mozilla.org/docs/Web/CSS/padding-block) for full RTL support
[`text-box: trim-both cap alphabetic`](https://developer.mozilla.org/docs/Web/CSS/text-box)

Precise vertical text alignment in legends
**Flexible option content**

Options use [flexbox](https://developer.mozilla.org/docs/Web/CSS/CSS_flexible_box_layout) with [gap](https://developer.mozilla.org/docs/Web/CSS/gap) for icon + text layout
**Checkmark auto-positioned**

[`order: 2`](https://developer.mozilla.org/docs/Web/CSS/order) and [`margin-inline-start: auto`](https://developer.mozilla.org/docs/Web/CSS/margin-inline-start) pushes checkmark to end of option

        
          Scrolling & Overflow
          [#](https://nerdy.dev/nice-select#scrolling-&-overflow)
        
       
[Scroll-state queries](https://developer.chrome.com/blog/css-scroll-state-queries) dynamically detect when content is scrollable and when sticky headers become stuck. Custom scrollbar styling and smooth scrolling enhance the browsing experience within long option lists.

[Container scroll-state queries](https://developer.mozilla.org/docs/Web/CSS/Guides/Conditional_rules/Container_scroll-state_queries)

[`container-type: scroll-state`](https://developer.mozilla.org/docs/Web/CSS/container-type) enables [`@container scroll-state(stuck)`](https://developer.mozilla.org/docs/Web/CSS/Guides/Conditional_rules/Container_scroll-state_queries#using_stuck_queries) detection
**Sticky legends**

Group headers [stick to top](https://developer.mozilla.org/docs/Web/CSS/position#sticky) when scrolling, with color change when stuck
**Custom scrollbar styling**

[`scrollbar-width: thin`](https://developer.mozilla.org/docs/Web/CSS/scrollbar-width) and [`scrollbar-color`](https://developer.mozilla.org/docs/Web/CSS/scrollbar-color) with fade on hover-out
**Scroll containment**

[`overscroll-behavior-block: contain`](https://developer.mozilla.org/docs/Web/CSS/overscroll-behavior-block) prevents scroll chaining
**Smooth scroll**

[`scroll-behavior: smooth`](https://developer.mozilla.org/docs/Web/CSS/scroll-behavior) for programmatic scrolling (motion-safe)
**Dynamic scrollable detection**

JS adds `.scrollable` class when content exceeds max height, CSS adjusts padding accordingly

        
          Customizable Select Features
          [#](https://nerdy.dev/nice-select#customizable-select-features)
        
       
New pseudo-elements provide surgical styling control over picker components, while superellipse corners deliver the modern aesthetic popularized by iOS. State selectors enable precise targeting of open and selected states.

[`::picker(select)`](https://developer.mozilla.org/docs/Web/CSS/::picker)

Pseudo-element for styling the dropdown picker container
[`::picker-icon`](https://developer.mozilla.org/docs/Web/CSS/::picker-icon)

Pseudo-element for the dropdown chevron indicator
[`::checkmark`](https://developer.mozilla.org/docs/Web/CSS/::checkmark)

Pseudo-element for the selected option indicator
[`<selectedcontent>`](https://developer.chrome.com/blog/customize-select#selectedcontent)

Native element that mirrors selected option content in the button
**Squircle corners**

[`corner-shape: superellipse(1.25)`](https://developer.mozilla.org/docs/Web/CSS/corner-shape) for modern rounded rectangles (with fallback)
[`:open` pseudo-class](https://developer.mozilla.org/docs/Web/CSS/:open)

State selector for when picker is visible
[`:checked`](https://developer.mozilla.org/docs/Web/CSS/:checked) on options

State selector for selected option

        
          Accessibility & UX
          [#](https://nerdy.dev/nice-select#accessibility-&-ux)
        
       
Accessibility is built-in with proper focus indicators, keyboard navigation, and semantic markup thanks to the web platform. Touch-friendly targets and overflow handling ensure the component works elegantly across all input methods.

**Focus ring styling**

Distinct [`:focus`](https://developer.mozilla.org/docs/Web/CSS/:focus) (subtle) and [`:focus-visible`](https://developer.mozilla.org/docs/Web/CSS/:focus-visible) (prominent)

[`option[disabled]`](https://developer.mozilla.org/docs/Web/HTML/Element/option#disabled) styling and hover prevention
**Text overflow handling**

[`text-overflow: ellipsis`](https://developer.mozilla.org/docs/Web/CSS/text-overflow) in button display
**Minimum touch target**

`36px` item height matches mobile guidelines
**Hidden option support**

JS skips [`[hidden]`](https://developer.mozilla.org/docs/Web/HTML/Global_attributes/hidden) options in offset calculations
**RTL example included**

Arabic language demo validates [bidirectional layout](https://developer.mozilla.org/docs/Web/CSS/CSS_writing_modes)

        
          Performance Optimizations
          [#](https://nerdy.dev/nice-select#performance-optimizations)
        
       
Careful optimization strategies keep animations smooth while minimizing layout thrash. Smart caching and compositor hints ensure the component performs well even on lower-end devices.

[`will-change: scale`](https://developer.mozilla.org/docs/Web/CSS/will-change)

Hints to browser for compositor optimization (also fixes text shift bug)
[WeakMap](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WeakMap) **offset cache**

Avoids expensive DOM measurements on every interaction;
auto-garbage-collected
**Hardcoded layout constants**

Avoids [`getComputedStyle`](https://developer.mozilla.org/docs/Web/API/Window/getComputedStyle) forced reflows during initialization. Tried dynamically computing these in a hidden instance, but couldn't get it as close as just "hardcoding / speficying them ahead of time" in a config
**Background clip**

[`background-clip: padding-box`](https://developer.mozilla.org/docs/Web/CSS/background-clip) prevents background bleeding under border
**Parallel icon variants**

Pre-defined black/white SVGs avoid runtime color manipulation

This select component intends to showcase what's possible when we combine modern CSS features with thoughtful progressive enhancement. The `appearance: base-select` property provides the foundation, while anchor positioning, scroll-state queries, and entrance animations create a polished, accessible experience.

The real power lies in how these features compose together‚Äîeach enhancement builds on platform primitives rather than fighting against them. The result is a component that's maintainable, accessible, and ready for the next generation of web interfaces.

Most importantly, this approach lets us create custom experiences without sacrificing the keyboard navigation, focus management, and screen reader support that come free with native elements.
