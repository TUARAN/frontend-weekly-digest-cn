# Form-Associated Custom Elements 实战

自定义元素这些年一直让人又爱又怕。爱的是它足够标准、足够原生、足够不依赖框架；怕的是一旦走进真实项目，就会碰到表单、校验、可访问性、序列化、焦点和集成体验这些麻烦问题。Form-Associated Custom Elements 的价值，正是在这里开始显现出来：它试图让自定义元素真正加入原生表单体系，而不是永远做个“看起来像控件”的外来者。

这篇文章值得看的地方，不是概念介绍，而是“它在实践里到底意味着什么”。当你的自定义元素可以参与表单提交、响应禁用状态、与校验逻辑协同后，很多过去必须靠额外胶水代码拼起来的集成就会自然得多。

这类能力真正动人的地方，在于它把组件抽象重新拉回平台。你不再需要在原生表单能力和组件封装之间二选一，而是开始有机会同时获得二者。对设计系统和跨框架组件库来说，这是一条很有潜力的路。

如果说过去的自定义元素更像“长得像表单控件的独立小岛”，那么 FACEs 想做的，是给这些岛真正接上一座桥。
