## 🗂 本期精选目录

### 🧭 Web 开发

* [Native JSON modules are finally real](https://allthingssmitty.com/2026/03/16/native-json-modules-are-finally-real/)：原生 JSON Modules 终于真正落地，前端模块系统又少了一块长期需要工具兜底的区域。
* [Building Seamless 3D Transitions with Webflow, GSAP, and Three.js](https://tympanus.net/codrops/2026/03/18/building-seamless-3d-transitions-with-webflow-gsap-and-three-js/)：把 Webflow、GSAP 与 Three.js 组合起来，做出非常顺滑的 3D 过渡体验。
* [WebMCP turns any Chrome web page into an MCP server for AI agents](https://thenewstack.io/webmcp-chrome-ai-agents/)：WebMCP 让网页与 AI Agent 的关系更进一步，浏览器页面开始能直接暴露 MCP 能力。
* [The 49MB Web Page](https://thatshubham.com/blog/news-audit)：一个极端但非常有教育意义的案例，展示“网页过重”到底会在现实里有多夸张。
* [Bun is Fast. Your Event Loop is Not.](https://howtocenterdiv.com/beyond-the-div/bun-is-fast-your-event-loop-is-not)：运行时再快，也救不了被阻塞的事件循环，这篇文章把性能讨论重新拉回根因。

#### 🛠 Tools

* [SVG Studio](https://www.svgstudio.org/)：一个浏览器内 SVG 动画编辑器，能把静态矢量图转成平滑关键帧动画并导出干净 SVG。
* [Tales from the Void: March Launch Week Recap](https://voidzero.dev/posts/whats-new-march-launch-week-2026)：VoidZero 三月 Launch Week 的整体回顾，适合补齐近期工具链方向感。
* [When Deno or Bun is a Better Solution than Node.js](https://frontendmasters.com/blog/when-deno-or-bun-is-a-better-solution-than-node-js/)：不是所有服务端 JS 都该默认上 Node，关键是明确场景和约束。

### 🎨 CSS

* [Dropdowns Inside Scrollable Containers: Why They Break And How To Fix Them Properly](https://www.smashingmagazine.com/2026/03/dropdowns-scrollable-containers-why-break-how-fix/)：滚动容器中的下拉菜单为什么总出问题，这篇文章把根因与正确修法讲得很透。
* [Image Gallery with Popovers and AIM (Anchor-Interpolated Morph)](https://frontendmasters.com/blog/image-gallery-with-popovers-and-aim-anchor-interpolated-morph/)：把 Popovers 和 Anchor-Interpolated Morph 组合起来，做出流畅的图片画廊过渡。
* [Flexbox Masonry Layout (Explained with Math)](https://frontendmasters.com/blog/flexbox-masonry-layout-explained-with-math/)：不用 JS 近似 Masonry 的经典命题，这次用数学方式讲清楚 Flexbox 解法。
* [More Easy Light-Dark Mode Switching: light-dark() is about to support images!](https://www.bram.us/2026/03/19/more-easy-light-dark-mode-switching-light-dark-is-about-to-support-images/)：`light-dark()` 即将从颜色扩展到图片资源，暗黑模式切换又少一堆手工逻辑。
* [Highlighting Footnotes](https://kittygiraudel.com/2026/03/18/highlighting-footnotes/)：一个小而美的体验优化话题：如何让脚注在阅读型页面里真正“被看见”。

### 💡 JavaScript

* [Temporal: The 9-Year Journey to Fix Time in JavaScript](https://bloomberg.github.io/js-blog/post/temporal/)：Temporal 的 9 年旅程本身，就是一段非常有代表性的语言演进史。
* [JavaScript for Everyone: Destructuring](https://css-tricks.com/javascript-for-everyone-destructuring/)：解构赋值看似基础，但它依然是写出更清晰 JS 的关键表达工具。
* [TC39 Advances Temporal to Stage 4 Alongside Several ECMAScript Proposals](https://socket.dev/blog/tc39-advances-temporal-to-stage-4)：Temporal 晋级 Stage 4 是一个重要节点，也说明 JavaScript 时间处理终于要进入新基线。
* [State of JavaScript 2025: Survey Reveals a Maturing Ecosystem with TypeScript Cementing Dominance](https://www.infoq.com/news/2026/03/state-of-js-survey-2025/)：JavaScript 生态正在成熟，而 TypeScript 的支配地位也越来越像共识。

#### ⚛️ React

* [Less code, more power: Why we rolled our own React Server Components framework](https://www.aha.io/engineering/articles/why-we-rolled-our-own-rsc-framework)：自己做 RSC 框架并不只是技术冒险，它通常反映的是团队对边界和复杂度的重新判断。
* [Two React Design Choices Developers Don’t Like — But Can’t Avoid](https://dev.to/playfulprogramming/two-react-design-choices-developers-dont-like-but-cant-avoid-d6g)：有些 React 设计长期被抱怨，但也恰恰构成了它今天的核心模型。
* [Why we banned React’s useEffect](https://x.com/alvinsng/article/2033969062834045089)：禁用 `useEffect` 这种激进做法背后，真正值得讨论的是副作用治理。
* [How We Used Next.js to Improve Core Web Vitals for SaaS](https://www.surajon.dev/nextjs-to-improve-core-web-vitals-for-saas)：把 Next.js 放回 SaaS 性能优化场景里，讨论真实的 Web Vitals 改善策略。
* [React SSR Framework Showdown: TanStack Start, React Router, and Next.js Under Load](https://blog.platformatic.dev/react-ssr-framework-benchmark-tanstack-start-react-router-nextjs)：用真实负载压测对比 React SSR 框架，是今年非常值得关注的实践型文章。

#### 🟢 Vue

* [State of Vue & Vite 2026: Amsterdam recap](https://laurentcazanove.com/blog/state-of-vue-vite-2026-amsterdam-recap)：一篇社区视角的 Vue & Vite 大会回顾，适合快速补齐生态方向感。
