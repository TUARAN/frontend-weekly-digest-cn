# 重新认识 JavaScript 的 .map() 方法

`.map()` 可能是 JavaScript 最常见也最容易被低估的方法之一。很多人学它时，只记住一句话：把数组映射成另一个数组。这个定义没错，但也太快结束了。真正用得好的开发者，会把 `.map()` 当成一种表达转换意图的方式，而不只是一个循环替代品。

这也是为什么类似“`.map()` 的几种典型用法”这种文章依然值得读。不是因为 API 本身很新，而是因为很多基础方法一旦进入复杂代码库，它们的使用方式就会慢慢反映出团队的表达习惯。写得清楚的人，用 `.map()` 会让数据流读起来非常顺；写得糟的人，则会把它和副作用、条件判断、空值处理缠在一起，让代码越来越难懂。

基础工具往往最能体现工程成熟度。不是因为它复杂，而是因为你会在最日常的地方暴露自己的思维习惯。`.map()` 就是这样的一个小镜子。

前端成长到某个阶段之后，真正重要的已经不是“我知不知道这个方法”，而是“我能不能用最恰当的方式表达这次转换”。
