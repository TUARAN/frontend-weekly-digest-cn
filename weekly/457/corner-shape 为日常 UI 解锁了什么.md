# corner-shape 为日常 UI 解锁了什么

前端过去十几年在“边角表达”这件事上，其实非常单调。绝大多数时候，大家的选择都只有方角和圆角，最多再通过不同数值做出一点性格差异。`border-radius` 足够伟大，但它也在很长时间里把边角美学限制在了一个相对狭窄的范围内。

`corner-shape` 之所以让人兴奋，不是因为它多了一个新玩具，而是因为它终于把边角从“数值大小问题”扩展成“形状语义问题”。按钮、卡片、标签、提示框、品牌组件，都开始有机会拥有更细微但更稳定的形体语言。

这类变化非常重要，因为现代 UI 的差异感很多时候并不来自更复杂的布局，而恰恰来自这些低频但持续存在的视觉信号。边角不是装饰细节，它常常是产品气质的一部分。

## 为什么这篇文章值得反复看

它的价值不只是在讲一个新属性，而是在展示原生 CSS 如何一步步接手过去要靠 SVG、额外伪元素甚至图形资产才能实现的形状表达。也就是说，CSS 正在变得更像一门真正能参与品牌和界面语言构建的系统，而不仅是给方盒子上色。

这会直接影响设计系统思路。以前很多团队会默认“组件边角差异做不了太多”，现在则开始可以把边角本身纳入系统变量与风格策略里。这种变化看起来很小，但对 UI 一致性和个性表达会有长期影响。

文章最值得带走的一点是：前端很多“设计味道”的提升，并不一定来自更重的图形方案，也可能来自 CSS 终于把基础形状表达做到了足够丰富。
