# React 有两个大家不喜欢却躲不开的设计选择

React 发展到今天，很多设计选择早已成为生态默认空气的一部分。可这并不意味着大家都喜欢它们。有些机制长期被抱怨、被吐槽、被拿来和其他框架比较，但最终大家还是得继续面对。因为这些设计恰恰构成了 React 心智模型的核心。

这类文章特别值得看，因为它不是在做“React 好不好”的浅层争论，而是在追问：为什么有些令人不适的设计，反而是这个框架始终没有放掉的部分？很多时候，一个框架真正的取舍价值，不在于它哪里最讨喜，而在于它愿意长期坚持哪些会让开发者不舒服、但能保持模型一致性的东西。

这也是理解 React 成熟路线的一条好入口。你越深入，会越发现它并不是总在追求“让人一眼就舒服”，而更像是在捍卫一套长期有效的抽象边界。代价当然存在，但边界也因此更稳定。

技术世界里很多让人“喜欢不起来”的设计，恰恰是最值得被认真理解的设计。
