💬 **推荐语**

这一期的内容，让我想起一句话：**越是基础的东西，越容易在关键时刻暴露出问题。**

Web Dev 栏目里，TanStack 的 npm 供应链攻击事件占据了整整两篇篇幅——一篇复盘，一篇加固方案。这不是某个边缘库的小事故，而是前端生态中依赖链条脆弱性的集中爆发。与此同时，HTML Sanitizer API、HTML-in-Canvas 提案、新的 install 元素这些平台级能力也在稳步推进，说明浏览器在安全边界和应用能力上的投入并未停止。

CSS 这边，gap decorations 终于稳定落地、`round()` 函数让流式尺寸更优雅、无限动画的精细化控制也补齐了最后一块拼图。这些能力叠加起来，CSS 正在从"能做什么"走向"怎么写更干净"。

JavaScript 栏目里 TypeScript v6 正式发布，ShadowRealm 提案让隔离执行有了新想象，而 Angular 也开始拥抱 WebMCP——工具链的进化从来不会等人。

React 这一期偏实战：父组件如何感知子组件变化、React Router 下对话框的边界处理、以及 React 应用安全的一篇系统性梳理。

如果这一期有一条暗线，那就是：**安全不是可选的附加项，平台能力不是锦上添花，工具链的进化也不是可慢可快的选择题。** 它们正在一起重新定义"前端工程师的基本功"。

## 🗂 本期精选目录

### 🧭 Web Dev

* [The HTML Sanitizer API](https://alfy.blog/2026/05/07/html-sanitizer-api.html)：浏览器原生 HTML 清洗 API 正在到来，第三方 sanitizer 库的统治可能要结束了。
* [Exploring the HTML-in-Canvas Proposal](https://tympanus.net/codrops/2026/05/13/exploring-the-html-in-canvas-proposal/)：把 HTML 画进 Canvas，不是截图，而是真正的可交互 DOM——这个提案一旦落地，可视化场景的玩法会彻底改变。
* [Postmortem: TanStack npm supply-chain compromise](https://tanstack.com/blog/npm-supply-chain-compromise-postmortem)：TanStack 被供应链攻击的完整复盘，值得每个团队当安全手册读一遍。
* [Install web apps with the new HTML install element](https://developer.chrome.com/blog/install-element-ot)：原生的 `<install>` 元素让 PWA 安装流程不再依赖 JS 触发，是渐进式 Web 应用体验的一块关键拼图。
* [From latency to instant: Modernizing GitHub Issues navigation performance](https://github.blog/engineering/architecture-optimization/from-latency-to-instant-modernizing-github-issues-navigation-performance/)：GitHub 如何把 Issues 页面的导航体验从"有延迟"做到"瞬间响应"，工程细节很值得学。
* [A Technical Deep Dive Into the New Raycast](https://www.raycast.com/blog/a-technical-deep-dive-into-the-new-raycast)：新 Raycast 的技术深潜，涉及 Electron 到原生渲染的架构迁移，桌面应用开发者的必读。
* [Hardening TanStack After the npm Compromise](https://tanstack.com/blog/incident-followup)：攻击发生后的加固方案，从发布流程到签名机制，是安全实践的活教材。
* [Vite+ guide: One CLI for JavaScript tooling](https://blog.logrocket.com/vite-plus-guide-cli-javascript-tooling/)：Vite+ 试图用一个 CLI 统一 JavaScript 工具链，思路很值得关注。
* [You don't know HTML Lists](https://blog.frankmtaylor.com/2026/05/13/you-dont-know-html-lists/)：你敢说你真的懂 HTML 列表？从语义到无障碍，这篇会让你重新认识这个最基础的元素。

### 🎨 CSS

* [How to Control Infinite CSS Animations (Part 2 of 2)](https://frontendmasters.com/blog/how-to-control-infinite-css-animations-part-2-of-2/)：无限动画的暂停、恢复、重置，这篇把控制权真正还给了开发者。
* [Gap decorations: Now available in Chromium](https://developer.chrome.com/blog/gap-decorations-stable)：gap 装饰线正式在 Chromium 稳定落地，Flexbox 和 Grid 的视觉分隔终于有了原生方案。
* [Repeating Square Dots Backgrounds in CSS](https://frontendmasters.com/blog/repeating-square-dots-backgrounds-in-css/)：用纯 CSS 实现重复方形点阵背景，把背景图案玩出了新花样。
* [Callout UI with CSS Offset & Border](https://frontendmasters.com/blog/callout-ui-with-css-offset-border/)：CSS offset + border 组合出的 Callout 标注组件，代码量少但效果惊艳。
* [Computing and Displaying Discounted Prices in CSS](https://css-tricks.com/computing-and-displaying-discounted-prices-in-css/)：用 CSS 计算并展示折扣价，把样式层的能力边界又往外推了一步。
* [Btw, syntax is well supported (Baseline)](https://bsky.app/profile/pawelgrzybek.com/post/3ml4nheptqc2z)：一句话提醒：很多你以为"还早"的 CSS 语法，其实已经是 Baseline 了——快速扫一眼，避免过度保守。
* [Better fluid sizing with round()](https://ishadeed.com/article/css-round/)：`round()` 让流式尺寸的计算不再产生小数像素的毛刺，是响应式排版的一个细腻升级。
* [How to Build Modern UI Components Using CSS Generators](https://www.sitepoint.com/how-to-build-modern-ui-components-using-css-generators/)：用 CSS Generator 工具快速构建现代 UI 组件，效率导向的实用指南。

### 💡 JavaScripts

* [Intentionally blocking rendering with JavaScript](https://www.jayfreestone.com/writing/intentional-render-blocking-javascript/)：故意用 JS 阻塞渲染？在某些场景下这反而是正确的性能策略，观点非常反直觉但有理有据。
* [TypeScript v6 is here: A full migration guide](https://blog.logrocket.com/typescript-v6-migration-guide/)：TypeScript v6 正式发布，这篇迁移指南覆盖了所有你需要知道的 breaking changes 和新特性。
* [Soon We Can Finally Banish JavaScript to the ShadowRealm](https://css-tricks.com/soon-we-can-finally-banish-javascript-to-the-shadowrealm/)：ShadowRealm 提案让 JS 代码可以在隔离环境中执行，第三方脚本安全又有了一条新路径。
* [Angular v22 WebMCP Tools Explained](https://briantree.se/angular-webmcp-tools/)：Angular v22 引入 WebMCP 工具，让 AI Agent 与 Angular 应用的交互有了标准化的入口。

### ⚛️ React

* [When React parent components need to know their children](https://www.jayfreestone.com/writing/updating-react-parents-in-response-to-changes-in-children/)：父组件何时需要感知子组件变化？这篇把"向上通信"的场景和方案梳理得很清楚。
* [The Modern React Stack Explained for 2026](https://jsdev.space/react-stack-2026/)：2026 年的现代 React 技术栈全景图，从路由到数据层到部署，适合做技术选型参考。
* [Untangling dialogs in React Router](https://programmingarehard.com/2026/05/06/react-router-dialogs.html/)：在 React Router 里处理对话框的边界——URL 状态、焦点管理和可访问性——讲得干净利落。
* [Security in React Applications](https://certificates.dev/blog/security-in-react-applications)：React 应用安全的系统性梳理，从 XSS 到 CSP，从依赖审计到服务端渲染的陷阱。

## 结语

第 466 期最值得记住的，不是某一篇单独的文章，而是两个并行的趋势：**平台安全在被迫升级，平台能力在主动进化。**

TanStack 的安全事件是一个警钟——npm 生态的信任链比你想象的脆弱。但好消息是，HTML Sanitizer API、ShadowRealm、CSP 的持续完善，正在把安全从"开发者自觉"变成"平台默认"。与此同时，CSS 的 gap decorations 和 `round()`、新的 `<install>` 元素、TypeScript v6，无一不在告诉你：**前端的基础设施正在变得更厚实、更可靠。**

越是基础的东西，越值得花时间掌握。这一期，就是这句话最好的注脚。
