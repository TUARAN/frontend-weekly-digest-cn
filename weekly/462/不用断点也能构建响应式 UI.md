# 不用断点也能构建响应式 UI

这篇文章很适合拿来重新校准我们对响应式设计的直觉。过去很多团队一提响应式，第一反应就是往样式里继续堆断点；但随着容器查询、弹性布局和更成熟的 CSS 原生能力出现，“按组件自然伸缩”已经越来越像默认方案，而不是高级技巧。

它真正有价值的地方，在于提醒我们：更好的响应式往往不是写更多条件分支，而是先把布局模型选对。断点当然不会消失，但它不该再是每个组件的第一解决手段。
