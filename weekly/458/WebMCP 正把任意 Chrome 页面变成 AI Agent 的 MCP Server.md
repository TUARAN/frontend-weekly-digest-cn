# WebMCP 正把任意 Chrome 页面变成 AI Agent 的 MCP Server

AI Agent 和浏览器之间的关系，正在从“自动操作页面”走向“页面本身主动暴露能力”。WebMCP 值得关注的地方，就在于它试图把普通网页从被动目标，变成一个可被 Agent 直接理解和调用的 MCP server 端点。

这件事的意义远不止浏览器自动化。过去 Agent 想和网页协作，往往只能像人一样点点点、读读读；而 WebMCP 代表的是另一条路：网页可以更明确地告诉 Agent 自己有哪些能力、哪些结构、哪些交互入口。这样一来，Web 和 Agent 的关系就不再只是“模拟操作”，而更接近“协议级协作”。

这对前端是个非常大的变化。因为它意味着网页未来可能不只是为人类视觉界面设计，也会越来越需要考虑自己如何作为一个“可供 Agent 消费的能力层”存在。语义、暴露方式、安全边界和权限控制，都会因此变成新的设计议题。

如果说过去的网页更像一块让人看的屏幕，那么 WebMCP 想推动的，是把它也变成一块能和机器对话的界面。
