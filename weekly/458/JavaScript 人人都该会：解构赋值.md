# JavaScript 人人都该会：解构赋值

解构赋值是那种“大家都会用，但不一定都用得明白”的 JavaScript 语法。它看起来足够基础，于是很多人学会了最常见的对象和数组解构之后，就再也不会回头认真看它。可一旦进入真实项目，你会发现解构的写法其实非常能反映一个人对数据结构和表达意图的把握。

用得好时，解构能让代码非常清楚：哪些字段是我真的关心的，哪些层级是我要明确展开的，哪些默认值是这里的语义边界。用得不好时，它也会让代码快速变得花哨、难读、隐藏依赖过多。也就是说，它不是“只是语法糖”，而是会影响代码叙事方式。

所以这类面向所有开发者的回顾文章依然有价值。不是因为大家不会写解构，而是因为真正的基础能力，恰恰最值得反复校准。很多代码风格上的成熟感，最后都来自这些最日常的语法有没有用在正确的位置上。

越基础的能力，越容易暴露出一个团队是否真的在追求清晰。
