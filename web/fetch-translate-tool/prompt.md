# 前端周刊翻译工作流

### 核心目标

* **翻译任务：** 将抓取到的英文前端技术文章进行逐段全量翻译，生成结构完整、技术准确、风格统一的中文翻译稿。
* **翻译要求：** 内容要自然、不生硬，避免机翻感，确保语言流畅且符合开发者的阅读习惯。
* **质量保证：** 逐段翻译，不做摘要或解读，确保翻译的每一个部分准确无误。

### 格式要求

* **图片格式：** 所有图片统一使用 `![]()` 的 Markdown 形式。优先使用外链图片，若外链不可用，保存图片并在文档中正确引用。
* **代码块：** 保留原文的代码块，不做修改，确保语法和结构一致。
* **段落顺序：** 保持原文的段落顺序不变，翻译时适度拆分长段落以提高可读性，但不改变含义。

### 接下来要做

1. **逐篇对照英文原文：** 继续调整“解读版”或“摘要版”的中文稿，确保它们符合“逐段全译”的要求。
2. **最终统一检查：**
   * 检查语气是否自然，是否有机翻感。
   * 确保图片格式正确、代码块完整。
   * 删除重复文件，确保最终提交的文档一致且没有遗漏。

### 工作流程

1. **抓取原文：** 使用脚本抓取英文文章。
2. **清理噪声：** 清理与正文无关的部分，如广告、推荐阅读、页脚等，保留正文内容和技术细节。
3. **逐段翻译：** 进行逐段全量翻译，确保术语准确，保留原文结构和技术细节。
4. **格式排版：** 统一文章格式，按《前端周刊》标准排版，确保文章易读且结构清晰。
5. **输出 Markdown：** 输出完整的 Markdown 文章，准备进入人工审阅或二次创作阶段。

### 文章初稿格式

```markdown
原文：{{原文标题}}
翻译：TUARAN
欢迎关注 {{前端周刊}}，每周更新国外论坛的前端热门文章，紧跟时事，掌握前端技术动态。

## {{文章大标题}}

{{中文翻译内容}}

```

---

请根据这些要求进行翻译，确保每篇文章按照流程处理，最终输出符合发布标准的高质量中文翻译稿。
