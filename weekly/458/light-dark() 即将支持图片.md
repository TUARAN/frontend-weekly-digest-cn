# light-dark() 即将支持图片

暗黑模式切换这些年最大的痛点，从来不只是颜色。颜色当然重要，但一旦产品里有大量插图、品牌资源、背景素材和图标资产，主题切换的复杂度就会迅速上升。`light-dark()` 如果开始支持图片，意味着这件事有机会从“资产管理问题”更进一步变成“样式表达问题”。

这很值得高兴，因为它再次体现了现代 CSS 一个非常明确的方向：把越来越多过去依赖额外脚本和资源切换逻辑的事情，重新收回到声明式层面。只要主题策略能更多写进 CSS，前端状态管理就会更轻，设计系统也会更清楚。

这种更新看起来像细节，其实会直接影响产品日常维护成本。一个大型站点如果要管理多套图片主题，过去常常需要一整套额外命名和切换逻辑；现在如果平台能接住一部分，系统会自然很多。

前端最舒服的演进，往往不是新功能突然多厉害，而是那些每天都要处理的小麻烦终于开始被平台照顾到了。
