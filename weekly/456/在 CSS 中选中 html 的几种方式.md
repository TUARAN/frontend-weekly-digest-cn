# 在 CSS 中选中 `html` 的几种方式：一个小知识点，为什么值得团队认真统一

前端里有一类问题，非常容易被轻视，因为它们看起来太小了。比如：在 CSS 里选中根元素，写 `html` 还是写 `:root`？很多人第一反应可能是，“这也值得讨论？” 但真正写过大型样式系统的人会知道，越是这种“好像都行”的地方，越容易在时间里慢慢长出混乱。

样式系统的维护成本，往往不来自那些复杂特效，而来自无数个微小但不统一的决策。今天这个文件用 `html`，明天那个主题变量挂在 `:root`，后天某个开发者再在根节点上加一个类名配合脚本切换。几年之后，大家会发现：谁都能看懂一点，但没人真正说得清整个系统为什么长成这样。

所以，讨论根元素选择器，并不是在抠字眼，而是在讨论样式系统的语言秩序。

## `html` 和 `:root`，表面一样，语义不完全一样

在 HTML 文档里，`:root` 选中的就是根元素，也就是 `<html>`。这也是为什么很多人会觉得它们没有区别。确实，在很多具体样式上，它们最终命中的对象完全相同。

但“命中相同”不等于“表达相同”。`html` 是一个具体元素选择器，它强调的是文档结构中的那个标签；`:root` 则更像一种语义选择器，它强调的是“当前文档树的根节点”。前者更贴近结构，后者更贴近系统层面的角色。

这点在 CSS 变量里尤其明显。很多团队习惯把全局设计 token 写在 `:root` 上，这其实是一种很自然的表达：这些变量属于整个文档的顶层作用域，而不只是“html 标签的一组样式”。

## 为什么小差异会影响长期维护

在一个小项目里，你也许完全可以混着写，几个月都不会出问题。但当项目开始变大、团队人数增加、主题系统和运行时状态介入后，这种“不统一”会慢慢放大成理解成本。

举个很常见的例子。如果团队约定 `:root` 用于全局 token 和主题变量，而 `html` 用于根级布局、滚动行为、字体缩放等文档样式，那么阅读一段 CSS 时，开发者几乎可以立刻推断意图。反过来，如果所有写法都混在一起，读代码的人就得自己猜：“这里是因为作者习惯这样写，还是因为它真的想表达结构而不是变量作用域？”

前端工程成熟的一个标志，不是有没有统一的代码风格工具，而是很多细小决定开始带着明确意图。

## 在主题系统里，这个区别尤其重要

现代前端项目越来越常见的一个模式，是通过给根元素添加属性或类名做主题切换，例如浅色/深色、高对比度、品牌皮肤、密度模式等。这个时候，根元素不仅是一个文档容器，也成了状态挂载点。

比如你可能会写：

```css
:root {
  --color-bg: white;
  --color-text: #111;
}

html[data-theme="dark"] {
  --color-bg: #111;
  --color-text: white;
}
```

这种写法就很有意思。默认 token 放在 `:root`，表达它们是系统级基线；运行时状态放在 `html[data-theme="dark"]`，表达“文档根节点此刻处于某种主题状态”。这时，两种选择器一起出现，不但不冲突，反而各自承担了不同职责。

## 一个实用的团队约定

如果你的团队现在还没有明确规范，我会建议用一个非常简单的约定来降低歧义。

第一，**` :root ` 主要用于定义全局 CSS 变量和系统 token**。第二，**`html` 主要用于根级布局与文档行为样式**，例如 `scroll-behavior`、`overflow`、基础字号、页面级状态类。第三，**运行时主题或模式切换，优先挂在 `html` 的类名或数据属性上**。

这套约定的好处是，不需要大家记一堆复杂规则，却能在读写样式时快速形成统一语感。写代码的人知道自己为什么这样写，读代码的人也更容易接住上下文。

工程上很多最有效的规范，往往都不是最复杂的，而是最容易长期执行的。

## 不要把 CSS 当成“反正浏览器能识别就行”

很多前端团队在 JavaScript 和 TypeScript 层面已经非常重视可读性与语义表达，但一到 CSS 层面，就容易退回“能跑就行”的习惯。可样式代码也是长期资产，同样需要被维护、重构、迁移和协作。

当你的项目有设计系统、有主题体系、有跨页面布局约束时，CSS 早就不只是“写点样式”，而是一套前端基础设施。基础设施最怕的不是复杂，而是没有命名秩序。

所以，像 `html` 和 `:root` 这种看起来很小的选择，其实正是建立秩序的起点。它们不决定产品功能，却会长期影响代码的可解释性。

## 写在最后

真正成熟的工程，不是把所有问题都处理成“大问题”，而是知道哪些小问题值得尽早统一。根元素选择器就是这种典型例子。它不会让你的页面瞬间变快，也不会替你解决业务难题，但它会在未来无数次阅读、修改和协作中，悄悄减少摩擦。

如果说样式系统像一种城市语言，那 `html` 和 `:root` 就像两个看似接近、其实语气不同的词。说对它们，不是为了显得专业，而是为了让整座城市的表达更加清楚。

有时候，技术成熟感并不来自用了多新的特性，而来自这些细节终于不再含糊。
