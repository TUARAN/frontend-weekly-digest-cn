# 地理位置元素的强制可访问性

很多前端开发者谈可访问性时，容易把它理解成一组“额外做好事”的清单：补 aria、补 label、补键盘交互、补语义信息。这当然重要，但有些平台能力正在走向另一条更值得注意的路：把可访问性直接嵌进能力本身，而不是留给开发者事后补齐。

地理位置相关元素的讨论，之所以有意思，就在于它展示了这种“被强制纳入”的趋势。也就是说，某些能力在设计之初就不再允许开发者完全忽略可访问性，而是把相关约束变成默认行为的一部分。

这对前端是个很好的提醒。真正成熟的平台，不应该把可访问性永远当作使用者的道德自觉，而应该尽可能把正确路径做成默认路径。因为现实是，默认值会塑造绝大多数产品行为。

所以这篇文章讨论的，不只是某个元素，而是一个更大的方向：未来越来越多 Web 能力，可能会把“可访问”从建议项变成内建属性。
