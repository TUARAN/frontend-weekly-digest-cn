# Eleventy 改名 Build Awesome

开源项目改名，表面上看像是品牌动作，实际往往反映的是项目对自身边界的重新定义。Eleventy 这次把叙事转向 Build Awesome，就很像是在告诉社区：我们不想只被理解成某一个静态站点工具，而是想被放进更大的“如何构建 Web 内容与体验”的语境里。

这类变化很有意思，因为它说明前端工具不再只是围绕“打包和渲染”竞争，也开始争夺叙事位置。你究竟是一个生成器、一个框架、一个内容引擎，还是一套更开放的构建哲学？名字往往只是外壳，真正变化的是项目希望社区如何理解它。

从 Eleventy 一路走来的发展看，它一直很强调内容优先、简单、可组合和不被复杂前端默认绑架。改名这件事如果放在这个脉络里，就不是脱离历史，而更像是在把原本隐含的价值观讲得更外放一点。

所以，与其只把这看成“Eleventy 换名字了”，不如把它看成前端内容工具生态的一次自我重申：在越来越重的应用框架之外，依然有人在认真维护一条更轻、更朴素、也更长期的构建路线。
