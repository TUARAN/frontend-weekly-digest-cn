# CSS Selection 2026：真实世界 CSS 使用现状

[原文链接：The CSS Selection (2026 edition)](https://www.projectwallace.com/the-css-selection/2026)

作者：Bart Veneman（Project Wallace）

这是一份“用数据看 CSS”的年度报告：作者分析了超过 100,000 个网站的真实 CSS，统计了文件体积、行数、复杂度、at-rule/选择器/声明的使用方式，以及一些新特性的采用情况。

原文包含大量图表与明细表格。这里做一份更适合周刊阅读的中文要点版，并保留关键数字与结论。

## 研究动机与方法概览

- 过去几年 Web Almanac 的 CSS 章节更新较少，而 CSS 近年变化极快。
- Project Wallace 使用自己的分析器做统计，因此和基于正则的分析结果可能会有差异。
- 样本规模是 100,000 个网站：不如 HTTP Archive 的“百万级”，但足以反映常见趋势。

## CSS 体量：普遍很大，但分布差异巨大

作者用“把 CSS 拼成一个连续的未压缩字符串”来衡量体积（更贴近“代码量”而非网络传输量）。

- 样式表体积分位数：
  - p10：32.6 kB
  - p25：130 kB
  - p50：309 kB
  - p75：619 kB
  - p90：1.12 MB
- 最大值（最夸张的网站）达到 52.5 MB（未压缩）。

这意味着：对相当一部分网站来说，CSS 已经是“成本很高”的资源。即使 gzip/br 压缩能显著降低传输体积，解析与计算样式仍然是一笔真实开销。

## 代码行数与复杂度

作者将“CSS 源码行数”定义为：

> source lines of code = #rules + #at-rules + #declarations

对应分位数：

- p10：1,029
- p25：4,551
- p50：10,677
- p75：20,013
- p90：33,457

复杂度（complexity）是作者定义的综合指标，用于衡量选择器复杂、浏览器 hack、前缀等因素带来的维护成本：

- p10：2,978
- p25：13,743
- p50：33,188
- p75：65,140
- p90：114,893

总体趋势很直观：当 CSS 体量增长时，复杂度往往也跟着增长。

## At-rule 使用概览：`@media` 仍是王者

在“采用率（有使用的站点占比）”上，最常见的 at-rule 依次是：

- `@media`：93.06%
- `@font-face`：85.62%
- `@keyframes`：83.90%
- `@supports`：44.57%
- `@charset`：39.57%
- `@import`：18.04%
- `@container`：9.61%
- `@layer`：2.71%
- `@property`：2.67%

其中两个值得注意的点：

- `@supports` 的采用率比很多人想象中低（不到一半的网站）。
- `@layer` 与 `@property` 虽然已可用一段时间，但在真实世界的普及仍然偏早期。

## 夸张的离群值（outliers）

报告里有不少“离谱但真实”的数据点，能提醒我们工程边界在哪里：

- 单页最多 at-rule：150,825
- 单条规则最多选择器：128,528（作者认为很多来自 Sass `@extend` 的滥用）

## 结论（作者想传达的核心）

- CSS 能力爆炸式增长，真实世界的 CSS 也在持续膨胀；
- 新能力（如 `@container`、`@layer`、`@property`）开始进入生产，但总体仍处于“早期采用”；
- 对多数团队来说，持续关注体量、复杂度与可维护性指标非常必要。
