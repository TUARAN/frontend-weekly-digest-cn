# Temporal：修好 JavaScript 时间的 9 年旅程

如果你写前端写得够久，就一定对时间处理有点复杂感情。`Date` 能用，但总让人不放心；Moment.js 曾经拯救了无数项目，但也逐渐显得笨重；各种时区、格式化、加减运算和语义混淆，总让“处理一个时间”听起来像小事，做起来却很容易出事故。Temporal 的 9 年旅程之所以值得回顾，正是因为它不是一个普通新 API，而是 JavaScript 社区试图系统性修好“时间”这个基础问题的过程。

这类语言级演进往往都很慢，因为它们改动的不是表面用法，而是根部模型。Temporal 真正要修的，并不只是让日期更方便，而是把过去被混在一起的概念重新拆清楚：绝对时间、本地时间、带时区的时间、纯日期、纯时间，它们原本就不该被一个模糊对象全都代替。

所以读这篇文章最有价值的，不只是补历史，而是理解为什么“修时间”会花 9 年。因为一旦你理解了这个过程，你就会更意识到：前端很多最痛的基础问题，从来不是没人想修，而是它们真的复杂到值得花这么长时间修。

Temporal 最终的意义，也许正是让 JavaScript 在一个长期被嘲笑的基础能力上，终于拥有了更像现代语言的答案。
