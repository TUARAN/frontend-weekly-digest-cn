💬 **推荐语**

这一期把「**HTML-first**」和「**安全默认值**」两件事讲到了极致。一边是 mohkohn 用 Astro + Web Components 把一家英国公用事业公司的表单完成率翻了一倍——上线才意识到：原来此前的 React 应用一直在用 20MB 的 JS 把老人机、PSP、3G 网络用户挡在门外；另一边是 Cloudflare 把 VoidZero 收入囊中、Rolldown 1.0 稳定、npm v12 默认关闭 postinstall 脚本——前端工具链的"地心引力"正从 JS 转向 Rust 与安全默认。

CSS 部分依然是 Bramus/Koen/master.dev 阵营的高产期：`@starting-style` + View Transitions + Popover 的 3-2-1 动画系统已经成熟到可以直接抄作业；View Transition Pseudo Tree、Grid Lanes、scrubbable staggered animation 把 CSS 推进到 2025 年没人敢想的"声明式交互动画"领域。**这一期是写给那些愿意重新审视"我到底需不需要 React / 我到底需不需要 async / 我到底需不需要 srcset"的开发者的。**

而最让我个人点头的，是 Matklad 的《CSS: Unavoidable Bad Parts》——他用过来人的语气告诉你：CSS 里有 80% 的"专业"内容其实是历史包袱，剩下 20% 才是真知识。如果你曾经被 margin 折叠折磨到深夜，或者在凌晨四点 debug 过一个莫名其妙的 viewport 高度问题，这篇就是你的治愈系读物。

## 结语

第 470 期的关键字是「**回归常识**」：HTML 优先、JS 可选、安全默认、声明式动画、HTML 优先于 srcset。如果你正在用 20MB 的 JS 渲染一个表单、或者为了一个同步函数硬塞 `async`、或者在写复杂的 `sizes` 属性——这一期就是来拯救你的。

下一期我们会持续追踪 Angular 22、React Server Components、Cloudflare 收购 VoidZero 之后 Vite 生态的走向。也欢迎在 GitHub 提 Issue 告诉我们你最想看的主题。
