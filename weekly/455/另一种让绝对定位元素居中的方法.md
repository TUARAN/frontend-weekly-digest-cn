## 另一种让（绝对定位）元素居中的方法

[layout](https://css-tricks.com/tag/layout/) [position](https://css-tricks.com/tag/position/)

![](https://secure.gravatar.com/avatar/165ac177dc80e530919963bbe0c419e5e05604ac178f5a7c64b0bcb67288d2a8?s=80&d=retro&r=pg)

发表于 Feb 27, 2026

TL;DR：我们可以用三行 CSS 让绝对定位元素居中。而且在所有浏览器上都能用！

```css
.element {
  position: absolute;
  place-self: center; 
  inset: 0;
}
```

为什么？这个问题就得展开说了。

近些年，CSS 引入了很多新特性：它们未必让我们能做“全新的事情”，但确实让很多事情变得更容易、更简单。比如，我们不再需要硬编码索引：

```html
<ul style="--t: 8">
  <li style="--i: 1"></li>
  <li style="--i: 2"></li>
  <!--  ...  -->
  <li style="--i: 8"></li>
</ul>
```

取而代之的是，这些都被浓缩进了 [`sibling-index()`](https://css-tricks.com/almanac/functions/s/sibling-index/) 和 [`sibling-count()`](https://css-tricks.com/almanac/functions/s/sibling-count/) 这两个函数里。类似的近期例子还有很多。

不过，有一件小事依然让人感觉我们几十年来都在重复同样的做法：**让一个绝对定位的元素居中**。我们通常会这么做：

```css
.element {
  position: absolute;
  top: 50%;
  left: 50%;
  
  translate: -50% -50%;
}
```

我们先把元素的左上角移动到中心点，然后再把它回退平移 50%，这样它就居中了。

（原文此处为 CodePen 演示，若无法显示请访问原文查看。）

这种方式没什么问题——我们几十年来一直这么写。但它仍然让人感觉像是 *老办法*。这就是 *唯一的办法* 吗？其实不是：还有一种不太为人熟知、但跨浏览器可用的方法，不仅能居中，还能更方便地放置任意绝对定位元素。更棒的是，它复用了我们熟悉的 [`align-self`](https://css-tricks.com/almanac/properties/a/align-self/) 和 [`justify-self`](https://css-tricks.com/almanac/properties/j/justify-self/) 属性。

事实证明，这些属性（以及它们的简写 `[place-self](https://css-tricks.com/almanac/properties/p/place-self/)`）现在也能作用于绝对定位元素了。不过，如果我们直接这样用，会发现元素连动都不动一下：

```css
/* Doesn't work!! */
.element {
  position: absolute;
  place-self: center; 
}
```

那么，`align-self` 和 `justify-self` 对绝对定位元素到底是怎么生效的？说它们“对齐元素”当然没错，但更具体地说：它们是把元素对齐在它的 **Inset-Modified Containing Block**（IMCB，插入值修改后的包含块）之内。好吧……那 IMCB 又是什么？

想象一下，我们把绝对定位元素的 `width` 和 `height` 都设为 `100%`。即便元素是绝对定位的，它当然也不会无限增长，而是会被一个称为 **containing block**（包含块）的东西所约束。

包含块是最近的、会创建新的 [stacking context（层叠上下文）](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Positioned_layout/Stacking_context) 的祖先元素。默认情况下，它是 `html` 元素。

我们可以用 `[inset](https://css-tricks.com/almanac/properties/i/inset/)` 相关属性（具体来说是 [`top`、`right`、`bottom`、`left`](https://css-tricks.com/almanac/properties/t/top-right-bottom-left/)）来修改这个包含块。我以前一直以为 `inset` 属性是在固定元素的四个角（我刚才几秒前甚至也这么说了），但实际上，在底层我们固定的是 IMCB 的 *边界*。

![](https://i0.wp.com/css-tricks.com/wp-content/uploads/2024/09/inset-modified-containing-block-example.png?resize=1920%2C1080&#038;ssl=1)

默认情况下，IMCB 的尺寸与元素自身的尺寸相同。因此在过去，[`align-self`](https://css-tricks.com/almanac/properties/a/align-self/) 和 [`justify-self`](https://css-tricks.com/almanac/properties/j/justify-self/) 试图把元素“居中到它自己里面”，结果当然什么都看不出来。接下来我们要做的最后一步，就是把 IMCB 设成与包含块一样大。

```css
.element {
  position: absolute;
  place-self: center; 
  top: 0;
  right: 0;
  bottom: 0;
  left: 0;
}
```

或者使用它们的 `[inset](https://css-tricks.com/almanac/properties/i/inset/)` 简写：

```css
.element {
  position: absolute;
  place-self: center; 
  inset: 0;
}
```

只要三行！对 CSS 迷来说这就是胜利。诚然，我这可能有点“作弊”，因为用老办法我们也可以用 `inset` 属性把它同样压缩到三行，但是……先暂时忽略这个事实吧。

（原文此处为 CodePen 演示，若无法显示请访问原文查看。）

我们并不局限于“居中”，因为其他所有的 `align-self` 和 `justify-self` 位置值也都能正常工作。这提供了一种更符合直觉/更地道的方式来定位绝对定位元素。

（原文此处为 CodePen 演示，若无法显示请访问原文查看。）

**小技巧：** 如果我们想让绝对定位元素与其包含块之间留出间距，可以给元素加 `margin`，或者把容器的 `inset` 设为期望的间距值。

更棒的是，我查了下 [Caniuse](https://caniuse.com/mdn-css_properties_place-self_position_absolute_context)。一开始看起来 Safari 似乎不支持，但实际测试后发现，它在所有浏览器上都能正常工作！
